1. 기준점의 차이
미디어 쿼리 (Media Query)
기준: 브라우저 전체의 뷰포트(화면 전체)의 크기를 기준으로 스타일을 적용합니다.
장점: 전체 레이아웃의 변경이 쉽습니다.
예를 들어, 화면이 커지면 레이아웃의 컬럼 수를 늘리거나 여백을 변경하는 식의 전체적인 디자인 조정이 가능합니다.

컨테이너 쿼리 (Container Query)
기준: 개별 부모 요소(컨테이너)의 크기를 기준으로 스타일을 적용합니다.
장점: 컴포넌트 단위로 반응형 디자인을 구현할 수 있습니다.
즉, 같은 컴포넌트라도 포함하고 있는 컨테이너의 크기에 따라 스타일이나 레이아웃이 달라지게 할 수 있어, 재사용성이 높아집니다.
==========================================================================================
2. 레이아웃 도구와의 비유 (Grid vs Flex)
Grid와 Flex

Grid: 전체 레이아웃의 구조를 잡을 때 주로 사용합니다.
Flex: 컴포넌트 내부의 요소들을 배치할 때 주로 사용합니다.
미디어 쿼리와 컨테이너 쿼리 비교

미디어 쿼리 ≈ Grid
전체적인 레이아웃을 제어하는 데 유리합니다.

컨테이너 쿼리 ≈ Flex
개별 컴포넌트의 내부 구조나 스타일을 세밀하게 조정할 수 있습니다.
==========================================================================================
3. 혼합 사용의 장점
유연성과 세밀함 증가

미디어 쿼리와 컨테이너 쿼리를 함께 사용하면, 브라우저 전체 크기에 따른 전체 레이아웃 변경과 동시에, 각 컴포넌트의 크기에 따라 개별 스타일을 조정할 수 있어 반응형 웹 구현이 더욱 유연해집니다.
예를 들어, 미디어 쿼리로 전체 페이지의 레이아웃(예: 그리드 시스템)을 설정하고, 컨테이너 쿼리로 개별 카드나 컴포넌트의 세부 스타일(예: 텍스트 크기, 배경색 등)을 조정할 수 있습니다.

결론
미디어 쿼리는 화면 전체의 크기에 반응하여 레이아웃을 조정하는 데 강력한 도구이고,
컨테이너 쿼리는 부모 요소의 크기를 기준으로 개별 컴포넌트 단위로 반응형 스타일을 적용할 수 있는 도구입니다.
